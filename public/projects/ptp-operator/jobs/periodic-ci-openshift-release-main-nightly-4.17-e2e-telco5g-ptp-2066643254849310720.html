Test 1: [e2e-telco5g-ptp] telco5g-ptp-tests

• Duration: ~8m33s
• Error: Error from server (NotFound): daemonsets.apps "linuxptp-daemon" not found
• Root Cause: The ptp-operator was deployed from branch release-4.17 and images built/pushed successfully, but the linuxptp-daemon DaemonSet was never created in openshift-ptp. The operator requires a PtpOperatorConfig CR to instantiate the daemon; the wait loop expired with a hard deadline only ~28 seconds out (1781568419→1781568447), indicating the deployment step never produced the DaemonSet. Likely the ptp-operator did not create the DaemonSet because no PTP-capable NICs were detected on nodes cnfdf30/31/32, so no matching PtpConfig was applied.
• Breaking PR/Commit: Unknown — infra-side condition; no single code change triggered this.
• Source File: https://github.com/openshift/ptp-operator/blob/release-4.17/hack/deploy.sh (deployment script embedded in CI step telco5g-ptp-tests)
• Is it a flake? yes — infrastructure-dependent; absence of PTP-capable hardware on the assigned baremetal nodes causes this transiently.
• Suggested Fix: Ensure the telco5g CI lab nodes (cnfdf30-32) have PTP-capable NICs assigned before the job runs; alternatively add a pre-flight hardware check and skip/fail-fast with a clear message rather than a silent timeout.
