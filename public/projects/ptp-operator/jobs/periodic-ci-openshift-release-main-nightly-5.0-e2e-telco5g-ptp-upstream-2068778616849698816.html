• Is it a flake? No for the pmc bug (deterministic code defect). The BC FREERUN component may be hardware-related.
• Suggested Fix: In getClockIDViaPMC, change []string{"pmc", "-b", "0", "-u", "-f", configFile, "GET", "PARENT_DATA_SET"} to []string{"pmc", "-b", "0", "-u", "-f", configFile, "GET PARENT_DATA_SET"} (single combined arg) so pmc receives the full command string as one argv element.

• Breaking PR/Commit: https://github.com/k8snetworkplumbingwg/ptp-operator/commit/3b93c9f4 (Jun 9, 2026 — "tests: add retries to PTP clock ID lookups")
• Source File: https://github.com/k8snetworkplumbingwg/ptp-operator/blob/main/test/pkg/ptphelper/ptphelper.go (line ~133)
• Is it a flake? No for the pmc bug (deterministic code defect). The BC FREERUN component may be hardware-related.
• Suggested Fix: In getClockIDViaPMC, change []string{"pmc", "-b", "0", "-u", "-f", configFile, "GET", "PARENT_DATA_SET"} to []string{"pmc", "-b", "0", "-u", "-f", configFile, "GET PARENT_DATA_SET"} (single combined arg) so pmc receives the full command string as one argv element.
