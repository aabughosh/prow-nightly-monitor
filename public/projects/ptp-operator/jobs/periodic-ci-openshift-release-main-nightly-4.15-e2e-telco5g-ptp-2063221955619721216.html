AI Analysis

Root Cause: Three PTP conformance tests failed in the BC and dualnicbc test modes on OCP 4.15. All failures involve clock-class verification during simulated upstream link loss:


1. Verify clockClass when locking PTP source on single NIC boundary clock (bc-serial): PMC correctly reported clockClass 248 after bringing ens2f3 down, but the node-level openshift_ptp_clock_class Prometheus metric stayed at 6 for 300s. On OCP < 4.18, cloud-event-proxy exposes only a node-level clock_class metric (no per-config label) that tracks the main clock via GetMainClockName(). Per-config freerun transitions are visible via PMC but not reflected in that aggregate metric. PR #210 added node-level metric assertions for OCP < 4.18 single-NIC BC, which is incorrect for freerun (248) checks.

2. BC clock class recovers to Locked after upstream link outage (bc-serial and dualnicbc-serial): After TurnAllPortsDown(), clock class never degraded from Locked (6). NetworkManager on RHCOS auto-recovers admin-down interfaces within ~4–7s, restoring sync before the test could observe degradation. PR #215 fixed this for TurnOffAndWaitFaulty() via nmcli managed=no, but TurnAllPortsDown()/TurnAllPortsUp() used by the recovery test were never updated.
