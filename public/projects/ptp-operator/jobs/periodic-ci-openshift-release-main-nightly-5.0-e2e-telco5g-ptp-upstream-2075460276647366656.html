Version: 5.0 | Total failures: 38



TL;DR

Two code regressions merged in late June broke metric label lookups and T-BC topology discovery, cascading to 38 failures.



Root Cause Groups


• ~36/38 failures — Interface alias mismatch in openshift_ptp_clock_state metric labels (primary driver across dualfollower and likely all remaining suites)

Upstream PR #617 (k8snetworkplumbingwg/linuxptp-daemon, commit fd6ff1e, merged 2026-06-30) introduced pkg/alias/manager.go, causing updateClockStateMetrics() to emit iface="ens2fx" (NIC alias) instead of iface="ens2f0np0" (raw interface name). All conformance tests that look up openshift_ptp_clock_state by raw interface name now find nothing, producing deterministic not found failures.


• 2/38 failures — T-BC topology solver finds no valid WPC NIC configuration (tbc + all suites)

PR #205 (openshift/ptp-operator, commit 84723c8/aede1e9, merged 2026-06-23) added PtpConfigTelcoBC() with strict solver constraints requiring WPC NICs (Intel E810 with GNSS/PPS) as both T-BC receiver and local grandmaster. The cnfdf30/31/32 lab hardware satisfies none of these constraints, so [SynchronizedBeforeSuite] aborts before any test body runs.

Actionable Recommendations


1. Fix metric label lookup in ptp-operator tests (unblocks ~36 failures).

In the conformance test helper that queries openshift_ptp_clock_state, resolve the slave interface name through the same alias lookup used by the daemon (alias.GetAliasValue(ifname)) before comparing metric labels. Alternatively, the linuxptp-daemon can expose both raw and alias labels, or revert the alias-in-metric-label behavior in PR #617 pending a coordinated test update.


2. Skip T-BC suite gracefully when hardware is absent (unblocks 2 failures).

Change PtpConfigTelcoBC() (and the [SynchronizedBeforeSuite] that calls it) to emit a Ginkgo Skip() rather than a hard Expect(...).To(BeNil()) error when solutions["TBC"] is empty. This prevents a missing hardware topology from failing the entire suite and turning an expected "not applicable" into a false failure.


3. Gate the T-BC test job on lab hardware capabilities.

Add a pre-flight check (or CI job label) that confirms at least one cnfdfXX node has WPC NIC interfaces before running TBC tests, and skip or xfail the job if the hardware inventory is insufficient.
