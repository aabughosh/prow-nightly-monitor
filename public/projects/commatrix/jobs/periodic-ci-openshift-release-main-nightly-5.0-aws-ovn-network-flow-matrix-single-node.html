TL;DR: CRI-O's ephemeral port (38919) now exposed after Linux dynamic-range filter removal in PR #400.


Root Cause Groups:


• Group 1: CRI-O ephemeral port no longer suppressed by dynamic-range filter (affects Tests: Validation [It] should validate the communication matrix ports match the node's open ports)
• Evidence:

From raw-ss-tcp: LISTEN 0 4096 10.0.71.48:38919 0.0.0.0:* users:(("crio",pid=4187,fd=11))


The failing test is at validation_test.go:161 — after calling filterOutPortsOfKnownServices (which skips rpc.statd, dnsmasq, dig but not crio) and filterOutPortsInDynamicRanges(missingEPSMat, commatrix.DynamicRanges), port 38919 survives both filters and causes the Fail(...) call.


• Root Cause: PR #400 / commit 7d56343 (Remove Linux Dynamic ranges from generated communication matrix) deleted getLinuxDynamicPrivateRange, which previously appended the Linux kernel ephemeral port range (typically 32768–60999, read from /proc/sys/net/ipv4/ip_local_port_range) to commatrix.DynamicRanges. CRI-O's streaming server binds to an ephemeral port on the node's external IP (38919 here, always in the Linux ephemeral range). Before PR #400, filterOutPortsInDynamicRanges would suppress it; now DynamicRanges only contains the Kubernetes NodePort range (~30000–32767), leaving 38919 unfiltered. The test fails at validation_test.go:161.

• Breaking PR/Commit: 7d56343 merged via PR #400 (https://github.com/openshift-kni/commatrix/pull/400), present on both main and release-5.0 branches.

• Is it a flake? No — crio always allocates an ephemeral port outside 30000–32767 on the node IP; this will fail consistently after PR #400 on every single-node run.

• Suggested Fix: Add crio to the filterOutPortsOfKnownServices function in validation_test.go (alongside rpc.statd), since CRI-O's streaming server port is an internal ephemeral port not intended to be documented in the communication matrix. Alternatively, reintroduce a targeted ephemeral-range filter for services known to use random ports.

• Affected tests: Validation [It] should validate the communication matrix ports match the node's open ports (validation_test.go:161)


Relation Between Groups: Single root cause — one PR removed the Linux ephemeral-range guard, exposing crio's random port.
