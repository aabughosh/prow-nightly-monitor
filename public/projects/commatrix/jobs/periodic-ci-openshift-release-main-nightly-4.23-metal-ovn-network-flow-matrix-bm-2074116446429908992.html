• Root Cause: Commit 7d56343 (PR #400, "Remove Linux Dynamic ranges from generated communication matrix", merged main Feb 5 2026, backported to release-4.23) removed the Linux ephemeral range (32768–60999) from the generated commatrix's DynamicRanges. The test at validation_test.go:159 calls filterOutPortsInDynamicRanges(missingEPSMat, commatrix.DynamicRanges) — but since DynamicRanges is now empty (no range entries in the generated CSV), the six crio ephemeral listening sockets (which always open in the ephemeral range) are not filtered out. filterOutPortsOfKnownServices (validation_test.go:221) only skips rpc.statd, dnsmasq, and dig — not crio — so all six ports reach Fail() at line 161.
• Breaking PR/Commit: [PR #400 / 7d56343](https://github.com/openshift-kni/commatrix/commit/7d56343e0d6e32f2fcc9b639ada47bb83cc57740) — "Remove Linux Dynamic ranges from generated communication matrix" (main); backport to release-4.23 is the proximate cause on this branch.
• Is it a flake? No — crio always opens ephemeral listener sockets; every run will hit this now that the dynamic-range filter is gone.
• Suggested Fix: Add "crio" to the filterOutPortsOfKnownServices function in test/e2e/validation_test.go alongside rpc.statd/dnsmasq. Alternatively, document the crio ephemeral range as a named dynamic range with Optional: true in the static entries (but filtering by service name in the test is cleaner and more targeted than re-adding the full ephemeral range).
• Affected tests: Validation [It] should validate the communication matrix ports match the node's open ports
