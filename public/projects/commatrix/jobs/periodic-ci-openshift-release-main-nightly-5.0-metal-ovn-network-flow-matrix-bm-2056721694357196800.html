Root cause: The nightly bare-metal network-flow-matrix job failed the spec Validation [It] should validate the communication matrix ports match the node's open ports because the documented matrix (docs/stable/raw/bm.csv for bare metal) no longer matched reality on OpenShift 5.0: several TCP rows described listeners that are not present on the nodes (listeners moved, were dropped, or never appear in ss), and UDP/68 was documented but not observed as in use. Separately, CRI-O listens on high ephemeral TCP ports on the node IP (e.g. 35693 on master-2); those sockets have no EndpointSlice by design. The test treats “open on node but no EndpointSlice” as an error unless the port is filtered or documented in a way the generator understands. filterOutPortsInDynamicRanges only covers the NodePort range (~30000–32767), not the Linux ephemeral range, so CRI-O stream ports were not excluded. rpc.statd noise is already skipped in filterOutPortsOfKnownServices.
