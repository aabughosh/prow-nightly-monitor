• Is it a flake? Yes — the cluster API server being unreachable is an infra transient; however the get_cluster_version() bug that turns an empty server version into the literal string "." is a latent code defect that turns any connectivity outage into a hard script failure instead of a graceful error or fallback.
• Suggested Fix: In get_cluster_version(), check whether version is empty and fall back to the T5CI_VERSION parameter (already available in the step env) before constructing testing_branch; also add set -e/error guard around oc version so the script exits cleanly with a clear message rather than silently continuing with an empty version.
