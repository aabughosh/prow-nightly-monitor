Root Cause: The e2e validation test "should validate the communication matrix ports match the node's open ports" compares EndpointSlice-derived matrix entries against actual listening sockets (ss). CRI-O binds stream-server listeners on each node's primary IP in the Linux ephemeral port range (32768–60999). These ports (34773, 44679, 45143 on masters; 37109, 37897, 40683 on workers) change on every reboot, have no Kubernetes EndpointSlice, and are not loopback-filtered because they listen on the node IP (e.g. 192.168.111.21). The test already skips similar ephemeral host daemons (rpc.statd, dnsmasq) via filterOutPortsOfKnownServices, but had no equivalent filter for CRI-O, causing a hard Fail(). The seven stale documented/EndpointSlice ports (6388, 8798×2, 9192, 9443, UDP 68×2) only produce warnings (+ in matrix-diff-ss / doc-diff) and did not cause this failure.
