• Duration: 1m43s
• Error: API server unreachable → get_cluster_version() returns . → git checkout release-. fails with pathspec 'release-.' did not match any file(s) known to git
• Evidence:

```

Unable to connect to the server: dial tcp 10.1.104.200:64425: i/o timeout

Cloning into '/tmp/secret/commatrix'...

/tmp/secret/commatrix /

error: pathspec 'release-.' did not match any file(s) known to git

{"component":"entrypoint","error":"wrapped process failed: exit status 1",...}

```

• Root Cause: The test script's get_cluster_version() calls oc version | grep "Server Version:" | awk '{print $3}'. Because the bare metal cluster API server at 10.1.104.200:64425 was unreachable (i/o timeout), oc version returns no server version — so major="", minor="", and echo "$major.$minor" returns . (a literal dot). This sets cluster_version="." and testing_branch="release-.", which does not exist in the repo. The cluster was healthy at setup completion (cnfdu18-master-0/1/2 all Ready, cluster version confirmed 4.21.0-0.nightly-2026-06-20-204857) but the API VIP became unreachable by the time the test pod launched — a transient bare metal network/infra failure.
• Breaking PR/Commit: Unknown — no code regression; the script robustness issue (no fallback to T5CI_VERSION=4.21 env var when server is unreachable) is a pre-existing fragility.
• Source File: [github.com/openshift-kni/commatrix/test/e2e/e2e_suite_test.go](https://github.com/openshift-kni/commatrix/blob/main/test/e2e/e2e_suite_test.go) — test step script inlined in pod spec (not in commatrix repo; lives in openshift/release step registry).
• Is it a flake? yes — API server was healthy at cluster-setup completion, became unreachable transiently; this is characteristic of a bare metal network blip.
• Suggested Fix: In the CI step script, fall back to the T5CI_VERSION env var when oc version cannot reach the server (e.g., cluster_version="${T5CI_VERSION:-$(get_cluster_version)}" with a non-empty guard). Separately, retry or alert on job infra when the API is unreachable at test-phase start.
