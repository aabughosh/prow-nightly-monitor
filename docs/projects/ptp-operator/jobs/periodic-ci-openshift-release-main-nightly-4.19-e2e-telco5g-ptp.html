• Root Cause: The ptp-operator/main test binary (compiled from TEST_BRANCH=main) calls BasicClockSyncCheck → GetProfileName → QualifyProfileName(config.Name, profile.Name), returning "test-slave1_test-slave1" (or "temp_temp" for test 3). This qualified name is passed directly to GetProfileLogID, which constructs the regex (?m).?Ptp4lConf: #profile: test-slave1_test-slave1(.|\n)?message_tag \[(.)\] (ptphelper.go:85, ptphelper.go:88). The linuxptp-daemon release-4.19 image, however, logs the unqualified profile name straight from the PtpProfile CRD field: output.profile_name = *nodeProfile.Name (daemon.go:1131), rendering to #profile: test-slave1 (config.go:437). The regex never matches; GetProfileLogID exhausts its 3-minute retry loop, BasicClockSyncCheck's outer Eventually times out, and all three tests fail at ~202 s.

QualifyProfileName (ptphelper.go:955):

```go

func QualifyProfileName(crName, profileName string) string {

prefix := crName + "_"

if strings.HasPrefix(profileName, prefix) { return profileName }

return prefix + profileName

}

```

GetProfileLogID (ptphelper.go:85):

```go

const logIDRegex = (?m).?Ptp4lConf: #profile: %s(.|\n)?message_tag \[(.*)\]

renderedRegex := fmt.Sprintf(logIDRegex, ptpConfigName) // ptpConfigName = "test-slave1_test-slave1"

```

linuxptp-daemon config.go:437:

```go

configOut = fmt.Sprintf("#profile: %s\n", conf.profile_name) // conf.profile_name = "test-slave1"
```

• Breaking PR/Commit: ptp-operator commit bdc9209 ("Enforce unique profile names per node across PtpConfig CRs"), merged to main only via [PR #178](https://github.com/openshift/ptp-operator/pull/178). No companion change was made to linuxptp-daemon (neither release-4.19 nor main) to log the crName_profileName format.
