With Port0 still SLAVE, Port1 never entered HOLDOVER. This is a hardware/infra failure — the dual-follower NIC on cnfdf32 (ens7f0/ens7f1) does not respond to software-initiated link-down as expected by the test.
• Breaking PR/Commit: Not a code regression. Most recent relevant change: [PR #224 ccd1e711](https://github.com/openshift/ptp-operator/commit/ccd1e711) (2026-04-20) replaced toggleNetworkInterface with PortEngine/TurnPortDown, which uses ip link set. The ip link set approach may not trigger a PTP-visible link-down on all NIC models.
• Source File: [https://github.com/openshift/ptp-operator/blob/release-5.0/test/conformance/serial/ptp.go#L756](https://github.com/openshift/ptp-operator/blob/release-5.0/test/conformance/serial/ptp.go#L756)
• Is it a flake? No — the port role never changed at all across 600s; this is a persistent hardware incompatibility, not a timing race.
• Suggested Fix: Verify that node cnfdf32 has PTP-capable NICs that respond to ip link set down with a PTP port state transition to FAULTY. If the NIC (or its driver) does not propagate link-down to the PTP layer, the CI node needs to be replaced or the hardware configuration corrected before this test can pass.

All 146 other tests passed; this is an isolated hardware/infra issue on the dual-follower test node.

Test 1: [dualfollower-serial] PTP e2e tests PTP ClockSync Dual follower can sync when one follower port goes down [Serial]

• Root Cause: The dual-follower topology requires two physical PTP-capable follower ports. • Breaking PR/Commit: Unknown — likely infra regression, not code; no ptp-operator source change implicated.
• Source File: https://github.com/k8snetworkplumbingwg/ptp-operator/blob/main/test/conformance/serial/ptp.go#L756
• Is it a flake? No — the 715s duration (close to full 10-min BasicClockSyncCheck timeout) indicates a genuine sync failure, not a transient race.
• Suggested Fix: Verify the physical NIC wiring and interface configuration for the dualfollower topology on nodes cnfdf30–cnfdf32. Check whether both follower ports are connected and the upstream PTP master is reachable on both.
