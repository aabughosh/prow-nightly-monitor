This is hit in loadbalancer.go:144 of the Cluster API AWS provider. Because the internal API-server NLB could not be created, the CAPI AWSCluster never reached Ready, the OpenShift installer timed out after 15 minutes, and the cluster was never provisioned — making all downstream gather steps fail with DNS-not-found errors against a non-existent API endpoint.
• Breaking PR/Commit: Unknown — this is an AWS account-level quota exhaustion, not a code change.
• Is it a flake? Yes — AWS quota limits are shared across concurrent CI jobs in the us-west-1 region; this is a transient capacity issue that resolves when other jobs clean up their load balancers.
• Suggested Fix: Retry the job (other CI runs will have released their LBs). If recurring, file a quota increase request for ELBv2 in us-west-1 for the openshift-org-aws-3 account, or configure the job to use an alternate region/account with headroom.
