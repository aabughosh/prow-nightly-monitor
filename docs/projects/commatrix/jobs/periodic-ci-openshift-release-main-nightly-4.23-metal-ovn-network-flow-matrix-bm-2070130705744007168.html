TL;DR: CRI-O ephemeral TCP ports lack EndpointSlice entries and are not filtered, causing the matrix-vs-ss validation test to fail.


Root Cause Groups:


• Group 1: CRI-O ephemeral TCP ports pass through all filters unblocked (affects Test: Validation [It] should validate the communication matrix ports match the node's open ports — test/e2e/validation_test.go:161)
• Evidence:

```

[FAILED] the following ports are used but don't have an endpointslice:

Ingress,TCP,40803,,crio,,,master,false

Ingress,TCP,40999,,crio,,,master,false

Ingress,TCP,44465,,crio,,,master,false

Ingress,TCP,39127,,crio,,,worker,false

Ingress,TCP,41873,,crio,,,worker,false

Ingress,TCP,44687,,crio,,,worker,false
```

From build-log.txt:180-187 and matrix-diff-ss (all six - prefixed crio entries).

• Root Cause: CRI-O binds ephemeral TCP ports (32768–60999 range) on both master and worker nodes at runtime; these have no Kubernetes EndpointSlice. Two filters protect against spurious failures: filterOutPortsOfKnownServices (excludes rpc.statd, dnsmasq, dig — but not crio) and filterOutPortsInDynamicRanges (excludes ports within commatrix.DynamicRanges — but commit 7d56343 intentionally removed the Linux ephemeral range from the generated commatrix). With both escape hatches closed for crio, the six ports reach the hard Fail() at validation_test.go:161. The first domino is that commit 6cf4613 added rpc.statd/dnsmasq/dig to the skip list but omitted crio.
• Breaking PR/Commit: Likely the same PR that introduced commit 6cf4613 ("Skip rpc.statd, dnsmasq and dig port in validation test"), which added the exclusion list in filterOutPortsOfKnownServices but missed the crio service. The removal of Linux ephemeral ranges (commit 7d56343 / fec55d) is the upstream enabler.
• Is it a flake? Partially — the specific port numbers change every run (ephemeral allocation), but CRI-O consistently opens several TCP ports in this range, so the failure reproduces reliably.
• Suggested Fix: Add crio to the skip list in filterOutPortsOfKnownServices (test/e2e/validation_test.go:221), mirroring the rpc.statd treatment. Alternatively, add crio ephemeral ports as static entries with a note that they are host-level and not backed by a Service.
• Affected tests: Validation [It] should validate the communication matrix ports match the node's open ports


Relation Between Groups: Single group. The 7 stale matrix entries (TCP 6388, 8798×2, 9192, 9443; UDP 68×2) are reported only as level=warning by filterOutPortsOfKnownServices's sibling code path and do not cause a hard failure; they are documentation debt but not the breaking issue.


Per-Version Notes: OCP 4.23 nightly (2026-06-24-155843) on baremetal OVN — CRI-O ephemeral port behavior is consistent with any recent 4.x release; this failure would affect any version where the Linux dynamic range was removed from the commatrix and crio was not added to the service skip list.
